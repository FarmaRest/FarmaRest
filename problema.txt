02
Crear el entorno virtual (venv)
El entorno virtual aísla las dependencias del proyecto. Ejecuta este comando estando dentro de la carpeta raíz del repositorio. Nunca instales paquetes globalmente.

Explorador
▾
📁 mi-proyecto
🐍 main.py
📄 requirements.txt
▸
📁 venv/  
← nuevo
⬛ bash
# Confirma que estás en la raíz del proyecto
~/mi-proyecto 
pwd
/home/usuario/mi-proyecto
# Crear el entorno virtual llamado "venv"
~/mi-proyecto 
python -m venv venv
# (sin output = creado correctamente ✓)
# Verificar que la carpeta fue creada
~/mi-proyecto 
ls -la
drwxr-xr-x venv/
-rw-r--r-- requirements.txt
-rw-r--r-- main.py
ℹ
Windows: si el comando python no funciona, usa py -m venv venv o python3 -m venv venv. La carpeta venv/ debe aparecer en el explorador de VS Code.

Consejo VS Code: abre la paleta de comandos con Ctrl+Shift+P y escribe Python: Select Interpreter, luego selecciona el intérprete dentro de ./venv. VS Code activará automáticamente el entorno en nuevas terminales.

06
Configurar las variables de entorno
Copia el archivo de ejemplo .env.example como .env y ajusta los valores según tu entorno local. Este archivo contiene la configuración de base de datos, secretos y parámetros del servidor.

⚠
Importante: el archivo .env está en el .gitignore y nunca debe subirse al repositorio. Solo .env.example se versiona. Cada estudiante configura su propio .env localmente.
